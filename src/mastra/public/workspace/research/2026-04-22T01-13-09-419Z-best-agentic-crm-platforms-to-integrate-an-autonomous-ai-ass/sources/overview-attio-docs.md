## Summary

Summary:

- Attio Developer Platform components:
  - App SDK: Embed TypeScript React apps in the Attio interface and run server functions with access to Attio’s REST API.
  - REST API: Read/write data in Attio workspaces, plus real-time event subscriptions via webhooks.
  - MCP (Manage Attio via AI): Connect AI tools (e.g., Claude, ChatGPT) to search, update, and manage the Attio workspace.

- Webhooks:
  - Webhook capability is available for real-time event notifications from the REST API, enabling event-driven automation and integrations.

- Custom objects and data model:
  - Attio supports custom objects to model domain-specific data beyond standard CRM entities, enabling flexible data schemas and relationships.

- Automation primitives:
  - Automation features allow building workflows and processes (likely via APIs and webhooks) to automate tasks within Attio and connected systems.

- Email-sync implementation:
  - Email synchronization is supported, enabling bidirectional email integration to link messages with Attio records and automate email-driven workflows.

- Endpoints and access:
  - REST API endpoints for CRUD operations on workspaces, objects, records, and associations.
  - GraphQL usage is mentioned in the developer context, suggesting GraphQL access alongside REST for querying, though the primary docs emphasize REST and event webhooks.
  - App SDK and REST API can be used together to create rich, integrated apps inside Attio and external systems.

- Getting started:
  - Start with the Overview to learn how to build Attio Apps, then explore guides, references, and code samples for deeper implementation details, including webhook setup, custom objects, and automation use cases.
